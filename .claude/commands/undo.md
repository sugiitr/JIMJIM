# /undo — 直前の変更を元に戻す

直前にClaudeが加えたコード変更・ファイル操作を安全に取り消す。

## 手順

1. `git diff HEAD` で直前の変更内容を確認する
2. 取り消すべき変更の範囲を特定する
3. ユーザーに確認してから `git checkout` または `git revert` を実行する
4. 変更が取り消されたことを確認する

## ルール

- **確認なしに実行しない**: 必ずユーザーに確認する
- `git stash` を使って一時退避も可能
- ファイル単位 / コミット単位のどちらでも対応
- `git revert` は履歴を残す安全な方法

## コマンド例

```bash
# ファイル単位で元に戻す
git checkout HEAD -- <ファイルパス>

# 直前のコミットを取り消す（履歴保持）
git revert HEAD

# ステージングを取り消す
git reset HEAD <ファイルパス>
```

$ARGUMENTS に取り消したいファイル名・コミットIDを渡してください（省略時は直前の変更）。
