# /clear — 作業スペースを整理する

不要なファイル・ビルド成果物・一時ファイルを安全に削除してクリーンな状態にする。

## 手順

1. 削除対象を特定する（.gitignoreに基づいて）
2. 削除前にリストをユーザーに確認する
3. 承認後に安全に削除する
4. `git status` でクリーンな状態を確認する

## 削除対象の例

```bash
# ビルド成果物
dist/, build/, .next/, .cache/

# 依存関係（再インストール可能なもの）
node_modules/

# 一時ファイル
*.log, *.tmp, .DS_Store

# テストカバレッジ
coverage/
```

## ルール

- **ソースコードは絶対に削除しない**
- `.env` ファイルは絶対に削除しない
- `git clean -n` でドライランを先に実行する
- 確認なしに削除を実行しない

## コマンド例

```bash
# 追跡されていないファイルの確認（削除しない）
git clean -n

# ビルド成果物のみ削除
git clean -fd --dry-run
```

$ARGUMENTS に削除したい対象（例: `dist`, `node_modules`, `logs`）を渡してください。
