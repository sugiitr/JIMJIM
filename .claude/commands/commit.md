# /commit — コミットメッセージを生成してコミットする

現在の変更内容（差分）に基づいて、分かりやすく適切なGitコミットメッセージを自動生成しコミットする。

## 手順

1. `git diff --staged` と `git status` で変更内容を確認する
2. 変更の種類を分類する（feat / fix / chore / refactor / docs / test）
3. 日本語で簡潔なコミットメッセージを生成する
4. ユーザーに確認してからコミットする

## フォーマット

```
<type>: <変更内容の要約>

<詳細（必要な場合）>
```

## ルール

- コミットメッセージは日本語で簡潔に
- `.env` や秘密情報が含まれていないか確認する
- ステージされていないファイルは含めない

$ARGUMENTS にコミット対象ファイルを渡せます（省略時は全ステージ済み変更）。
