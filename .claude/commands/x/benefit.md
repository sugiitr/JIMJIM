# /x/benefit — ベネフィットを最大化した投稿を作る

商品・サービス・ノウハウの「機能」ではなく「変化・恩恵」を伝える投稿を生成する。

## 手順

1. 商品・機能・ターゲットを確認する
2. 機能→ベネフィット→感情的ベネフィットに変換する
3. 「〜できる」「〜になれる」の言葉で投稿を生成する
4. before/afterの対比で変化を可視化する

## 機能 → ベネフィット変換の例

```
機能: 「24時間サポート」
↓ベネフィット: 「深夜でも即解決できる」
↓感情的ベネフィット: 「不安なく眠れる」

機能: 「自動化ツール」
↓ベネフィット: 「作業時間が3時間→15分に」
↓感情的ベネフィット: 「家族との時間が増える」
```

## 出力形式

```
## ベネフィット投稿: 「〜」

### 変換テーブル
| 機能 | ベネフィット | 感情的ベネフィット |
|------|------------|-----------------|
| 〜 | 〜 | 〜 |

### 投稿案
〜

### Before/After版
Before: 〜
After: 〜
```

$ARGUMENTS に商品・機能・ターゲットを渡してください。
