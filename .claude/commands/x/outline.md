# /x/outline — コンテンツの構成を設計する

投稿・スレッド・ノート・記事の構成（アウトライン）を設計する。

## 手順

1. テーマ・形式・ターゲット・目的を確認する
2. 最適な構成パターンを選択する
3. アウトラインを生成する
4. 各セクションの要点と書き方ヒントを添える

## 構成パターン

- **PREP法**: 結論→理由→例→結論
- **問題解決型**: 悩み→原因→解決策→行動
- **ストーリー型**: 状況→葛藤→転換→結末
- **リスト型**: 概要→項目1〜N→まとめ
- **教育型**: 概念→説明→例→応用

## 出力形式

```
## アウトライン: 「〜」

### 使用構成: 〜型
### 目標文字数/ツイート数: 〜

---
## 1. 〜
- ポイント: 〜
- 書き方ヒント: 〜

## 2. 〜
...
```

$ARGUMENTS にテーマ・形式（ツイート/スレッド/ノート）・目的を渡してください。
